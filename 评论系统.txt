功能定位
消息级讨论空间：每条 Telegram 消息都可以生成一个独立的讨论空间

轻量级：不是完整聊天室，只是消息的“扩展对话区”

容量限制：最多 10 人同时在线参与

显示方式：侧边栏/弹窗，不离开当前主界面

与现有功能的区别
功能	传统回复	你的讨论空间
结构	树形嵌套	扁平化流式
人数	无限制	最多10人同时
显示	内联展开	独立侧边栏/弹窗
性质	永久存储	可设定临时/永久
互动	单向回复	多人实时聊天
二、产品功能设计

在每条消息下方增加第三个按钮：

3.2 实时通信架构
text
推荐方案：WebSocket + Redis

用户进入空间 → 建立 WebSocket 连接
↓
加入 Redis Pub/Sub 频道：space:msg_12345_space
↓
新消息发布 → 广播给所有在线用户
↓
用户离开 → 清理连接，更新在线人数

// 加入空间
async function joinSpace(spaceId, userId) {
  const space = await getSpace(spaceId);
  const currentParticipants = await getOnlineCount(spaceId);
  
  if (currentParticipants >= space.maxParticipants) {
    // 方案A：拒绝加入
    return { success: false, reason: "room_full" };
    
    // 方案B：踢出最不活跃用户（可选）
    // const inactiveUser = await kickInactiveUser(spaceId);
    // return { success: true, kickedUser: inactiveUser };
  }
  
  // 检查用户是否已在其他空间（限制每人同时只能在一个空间）
  // 可选功能
  
  await addParticipant(spaceId, userId);
  broadcastSystemMessage(`${userId} 加入了空间`);
  
  return { success: true };
}
